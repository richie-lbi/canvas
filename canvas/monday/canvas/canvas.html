A 'no canvas' fall back sits here
  1. Single DOM element - a container
  2. Requires JavaScript
  3. Contexts - getContext() - returns object with drawing methods, 3D context is called WebGL
  4. Bitmaps, Games, Animations, High performance operations like filters and shadows, animations
  5. Required </canvas> tag
  6. Take advantage of the Client not server - Load the raw data from the server but generate graphics on the Client
  7. Beware of the loaded image - CORS tainted canvas
  8. Why use a library?
  9. The 3 libraries are: EaselJS, KineticJS and PaperJS
